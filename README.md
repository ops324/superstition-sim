# 迷信の誕生シミュレーション（スキナーのハト実験のAI版）

報酬を**行動と無関係にランダム配布**しているだけなのに、学習エージェントが偶然の初期ゆらぎから**誤った因果信念（迷信）**を作り上げていく様子を観察する、単一 HTML のインタラクティブ・ダッシュボードです。B.F. スキナーのハトの「迷信行動」実験（1948）のAI版。

- **ビルド不要・外部依存なし・完全オフライン動作**（`index.html` を開くだけ）
- UI・解説はすべて日本語

> 📊 開発状況: 本リポジトリは **PRベースで直列に実装中**です。現在は **PR④（描画コア＋UI）** まで実装済み（PRNG・報酬環境・構造監査・エージェントA/B・シミュレーションエンジン・過信度／信念集中度・統計エンジン・母集団Canvasパネル・過信度時系列・操作系・再生/一時停止・初見者向けUI）。`index.html` をブラウザで開くと自動でシミュレーションが動作します。ペルソナ／多数回試行等は後続PRで追加します（末尾「実装ロードマップ」参照）。

## 何を見せるのか

各タイムステップで報酬 `r` は確率 `p` の**ベルヌーイ試行**で発生し、**エージェントがどの行動を選んだかとは完全に無関係**です。にもかかわらず、「直前の行動が報酬を生んだ」と学習してしまうと、その行動が選ばれやすくなり、さらに強化される——という正のフィードバックだけで、**根拠のない「儀式（迷信）」が生まれます**。

2種類のエージェントを同じ環境・同じ選択規則で並べ、**更新則だけ**を変えて比較します。

| | A. 強化エージェント（行動主義） | B. 確証バイアス・エージェント（認知） |
|---|---|---|
| 更新 | 対称 `V[a] += α·(r − V[a])` | 非対称（当たり重く・外れ軽く） |
| 解釈 | 当たりも外れも等しく学ぶ | うまくいった時ばかり覚え、外した時は忘れる |
| β=0 のとき | — | **Aと完全に一致** |

問い: **「確証バイアス（β）だけを足すと、公平な強化より迷信は強まるのか？」** を、β という1変数だけ変えて実験します（β=0 で B≡A なので、差は純粋に β に帰属＝クリーンな対照実験）。

### B の非対称更新（確証バイアス）
選んだ行動 `a` について:

- 報酬あり（確証）: `V[a] += αp·(1 − V[a])`,  `αp = min(α·(1+β), 1)`
- 報酬なし（反証）: `V[a] += αn·(0 − V[a])`,  `αn = α·(1−β)`

信念の平衡点は `V* = p(1+β) / [ p(1+β) + (1−p)(1−β) ]`。例えば `p=0.3, β=0.5` で **V*≈0.56**、`β=0.9` で **V*≈0.89** と、真の報酬率 0.3 を大きく超えて**信念が膨張**します。この過信が「その行動は効く」という誤った確信を生み、特定の行動を"儀式"に変えます。

### 「迷信の強さ」の指標＝過信度（over-belief）
この過信こそが迷信の本体です。指標は **過信度 `overBelief = clamp((max(V) − p) / (1 − p), 0, 1)`**：いちばん信じている行動（＝儀式）の期待値が、真の報酬率 `p` をどれだけ超えて膨らんでいるか。

- **0** ＝ 現実どおりに評価（迷信なし）、**1** ＝「絶対に効く」と確信（最大の迷信）。
- **A班**は対称更新で真の `p` に収束するため **≈0**（迷信なし）。**B班**は確証バイアスで `p` を超えて膨張し **≫0**（強い迷信）。この差が「思い込みが迷信を生む」ことの直接の証拠。

> ⚠️ **設計変更ノート**: 当初は「迷信の強さ」を**信念集中度（正規化 Herfindahl）**で測る設計でしたが、実エンジン検証で、確証バイアス型Bは探索により**全行動を一律に過信する**ため信念が1つに集中せず、集中度ではむしろ B < A となり中心命題を示せないことが判明しました。そこで中心指標を**過信度（真の p からの乖離）**に変更しています（集中度は「儀式が1つに絞られたか」の副指標として保持）。

## 「恣意性を排する」ための担保

このツールの核心は、**見えている迷信が本当に虚偽であること（報酬は行動と無関係であること）を、設計者の恣意ではなく構造で保証する**ことです。

### 1. 構造監査（第一の担保）
報酬生成関数 `createRewardStream(p, seed)` は、**引数にもクロージャにも「行動」変数を一切持ちません**。したがって報酬は行動を参照しようがありません。さらにシミュレーション1ステップは、必ず

```
r を先に確定  →  その後に行動 a を選ぶ  →  更新
```

の順で実行されます（`stepSimulation` 内、[index.html](index.html) の該当セクション参照）。「行動を読めないことがコード上自明」であることが第一の保証です。

### 2. 較正（第二の担保）
「報酬率は行動に依存しない」という帰無仮説の検定 p 値を、多数の乱数シードで集めると **一様分布 Uniform[0,1]** になります（正しく較正され、偽陽性率が約5%）。単一の検定結果を「独立の証明」として掲げることはしません——**統計的に「棄却できない」ことは「証明」ではない**からです。表現は一貫して「**独立と整合的（consistent with independence）**」とします。

### 3. 一様性（第三の担保）
多数回の独立試行で、**どの行動が優勢（＝迷信の対象）になるかは行動間で一様**に分布します。特定の行動がコード上えこひいきされていないことの検証です（タイブレークのランダム化などの回帰ガードでもあります）。

### 設計者の選択を隠さない
`V0`（初期値）・更新則の非対称性・選択規則・`β`・ペルソナ特性は**すべて画面上の可視スライダー/表示**にしてあり、「β=0 で B≡A」「V0>p で迷信が消える」などを自分で操作して確かめられます。

## 主なパラメータ（デフォルト）

| パラメータ | 既定 | 意味 |
|---|---|---|
| N | 5 | 行動の種類数 |
| p | 0.3 | 各ステップの報酬確率（行動と無関係） |
| α | 0.1 | 学習率 |
| V0 | 0.0 | 価値の初期値（悲観的。`V0>p` で迷信が消えることも実演可） |
| ε | 0.1 | 探索率（epsilon-greedy、両エージェント共通） |
| β | 0.5 | 確証バイアス（B のみ。0 で B≡A） |
| T | 3000 | ステップ数 |
| seed | 12345 | 乱数シード（再現性） |
| K | 100 | 多数回試行の本数 |

## 使い方（実装完了後）

`index.html` をブラウザで開くだけです（開いた瞬間に自動で再生が始まります）。`<script type="module">` を使っていないため、`file://` での直接オープンでも動作します。キャッシュに注意する場合はローカル静的サーバ（例 `python3 -m http.server`）経由でも確認できます。

**画面上の用語について（初見者向けの言い換え）**: UI では専門用語を平易な日本語で表記しています — 「迷信の強さ／過信度」＝真の報酬率 p を超えて信じ込む度合い `(max(V)−p)/(1−p)`、「儀式」＝いちばん信じている行動、「エサ」＝報酬、「A班＝公平に学ぶ / B班＝当たりだけ強く覚える」＝対称更新 / 確証バイアス非対称更新、「学習の速さ」＝α、「気まぐれ度」＝ε、「思い込み」＝β。時系列グラフの点線は真の p（現実）で、B班の線がこれを大きく超える＝過信＝迷信。数式・監査の詳細は画面下部の「詳しい仕組み」折りたたみと本 README を参照してください。

## 既知の限界

- 独立性検定の「非有意」は **独立の証明ではなく、独立と整合的というだけ**です（本文参照）。統計的確証は多数回試行の p 値較正・一様性が担います。
- **過信度（中心指標）は持続的**です（B班は `V*` へ膨張し p を超え続ける）。一方、迷信が**単一の"儀式"に固着するか（信念集中度）は探索 ε に依存**し、ε が大きいと B班は全行動を一律に過信して集中度は下がります（＝「何をやっても効く」型の拡散した迷信）。`V0>p`（楽観的初期値）は特に A班の過信を抑え、迷信が生まれにくくなります。強さ・持続性はパラメータ（`β`, `ε`, `V0` と `p` の大小, `α`）に依存します。
